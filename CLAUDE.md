If the stream ends after `message_start` but before any `content_block_start` — refusal, `stop_reason: max_tokens` at 0 continuation tokens, or stream-cut mid-flight (network blip / CF Workers subrequest edge timing) — `finalMessage()` returns a Message with `content: []`. The SDK does NOT synthesise a fallback. The TypeScript type `content: Array<ContentBlock>` permits empty arrays. Non-streaming `messages.create` can produce the same shape for refusals. So `response.content[0]` is structurally allowed to be `undefined`. **Which trigger fired on the tick-borne piece is not knowable from code alone** — refusal, max-tokens-at-zero, or stream-cut all produce the identical crash with no diagnostic. The HTML paths in `interactive-generator.ts` are the most-exposed sites because they use `.stream(...).finalMessage()` on the longest-running calls (4.2-7.2k tokens, 57-156s wall-clock per the 2026-05-09 streaming-rollout entry); the streaming-cut surface is widest there. ## The Daylila Protocol

**"Educate myself for humble decisions."**

"Most human suffering — personal, in organisations, and across the world — comes from treating connected things as if they were separate. The cure is learning to see and work with the whole."

Everything that follows is an attempt to show you what that means — and how to do it.

## What Daylila v2 is

An autonomous multi-agent publishing system. 16 AI agents scan the news, decide what to teach, draft pieces, audit them through quality gates, categorise them into the library taxonomy, generate a standalone quiz per piece, and publish — all without human intervention. Readers see a daily teaching piece anchored in today's news, with a growing library of past pieces and an optional quiz at the end of each.

### The 16 Agents (one job per agent, one file per agent)

Pipeline: Scanner → Curator → Drafter → [Voice, Structure, Fact] → Integrator → Publisher → Audio Producer → Audio Auditor → Publisher.publishAudio (second commit splices audioBeats into frontmatter). Text ships first — audio is ship-and-retry so the day is never blank. Observer receives events throughout. Learner + Categoriser + InteractiveGenerator (which internally calls InteractiveAuditor) run off-pipeline post-publish.

1. **ScannerAgent** — fetches 10 RSS feeds (6 Google News topic feeds + 4 breadth feeds covering entertainment / sports / food / personal-finance, since PR #1 2026-05-09 replaced the earlier 11 academic-leaning feeds), deduplicates, stores up to 24 candidates per run. Per-feed cap 2 (since 2026-05-11), global cap 24 (since 2026-05-10).
2. **DirectorAgent** — pure orchestrator. Routes work between agents. Zero LLM calls. Hourly cron gated by `admin_settings.interval_hours` (default 24 → fires at 02:00 UTC once per day).
3. **CuratorAgent** — picks the most teachable story from today's candidates, plans beats + hook + teaching angle. Reframed 2026-04-25 around the Daylila Protocol (no "60+ teachability threshold" gate); diversity-tuned 2026-05-01 with a 10-domain breadth taxonomy + recent-category-concentration block. As of 2026-05-06 (Foundation Fix Task 03) Curator records `pick_reasoning` on the picked candidate and `rejection_category` (closed 8-value enum: `off_topic`, `duplicate`, `too_local`, `no_teaching_angle`, `wrong_shape`, `low_signal`, `tribal_framing`, `already_covered`) + `rejection_reason` (top-5 runner-ups only) on every rejection — closes data leaks L1 + L2. `max_tokens` raised 3000 → 8000 to fit the new response shape.
4. **DrafterAgent** — writes the MDX from the brief, enforces `<lesson-shell>` / `<lesson-beat>` format. Reads recent learnings at runtime; self-reflects post-publish. Returns `loadedLearningIds` in `DrafterResult` so Director can attribute the new piece to those learnings on publish (Foundation Fix Task 04). As of 2026-05-07 (Foundation Fix Task 06) also accepts `pieceId` and writes round 0 of the audit-revise loop (the initial draft MDX) to `draft_revisions` (migration 0034) via `persistInitialDraft()`. Persistence is fail-open via `persistError` sentinel returned in `DrafterResult`; Director logs once via `observer.logError`.
5. **VoiceAuditorAgent** — voice compliance gate (≥85/100)
6. **FactCheckerAgent** — verifies every claim (single-pass: Claude with Anthropic `web_search_20250305` server tool; today's date in user message; search-first for current-event claims). Reads `content/fact-check-contract.md` via `${FACT_CHECK_CONTRACT}` injection since 2026-05-07 — verdict taxonomy + search-first rule + cutoff-confession ban + `max_uses=8` budget all live there. Path A.1 (2026-05-01) harvests citation URLs from `web_search_tool_result.content[]` for the drawer's "Sources consulted" line.
7. **StructureEditorAgent** — reviews flow and pacing
8. **IntegratorAgent** — handles revisions before approval (3 rounds max). As of 2026-05-07 (Foundation Fix Task 06) returns `{ revisedMdx, decisions[], parseError, persistError }` (was `{ revisedMdx, changesSummary }`); the decisions array carries one `IntegratorDecisionRecord` per feedback item the Integrator addressed, with closed-enum `decision` (`accepted` | `overruled` | `partial`) and `feedback_source` (`voice_auditor` | `fact_checker` | `structure_editor`). Each `revise()` call also persists one row to `draft_revisions` (round N) plus one row per decision to `integrator_decisions` (migration 0034) via `persistRevision()`. Both writes happen in a single `this.env.DB.batch()`. Parse-fail fallback preserves `revisedMdx` (raw text) with `decisions: []` + `parseError` populated; persistence is fail-open via `persistError` sentinel — same posture as Audio Auditor's persistError pattern (Task 05). Director logs both sentinels once via `observer.logError`. Reads `${INTEGRATOR_CONTRACT}` from `content/integrator-contract.md` at the system-prompt position. **Round-aware within a piece since 2026-05-07 (Foundation Fix Phase 4 Task 09):** extends `Agent<Env, IntegratorState>` with `initialState: { lastSnapshot: null }`. The user message always contains all three audit sections (PASS or FAIL, with PRESERVE/FIX framing — passing dimensions get an explicit "do not introduce changes that would re-break this" instruction); when DO state holds a same-pieceId snapshot from the previous round, a `## Previous round (round N-1) audit summary` block prepends the message so pass→fail flips across rounds register as regressions to repair. State read is keyed by piece_id with lazy reset (mismatched piece_id → ignored on read, overwritten on write); state write is unconditional regardless of parseError/persistError. Two new pure helpers `buildCurrentRoundFeedback` and `buildPreviousRoundContext` exported from `integrator.ts` for unit-test reach. Closes the [open] 2026-05-12 FOLLOWUPS entry "Integrator regression risk." Operator queries at `scripts/integrator-regression-health.sql`; 30-day post-deploy regression-rate evaluation queued as FOLLOWUPS [observing] entry.
9. **AudioProducerAgent** — generates per-beat MP3 via ElevenLabs (Frederick Surrey, `eleven_multilingual_v2`, 96 kbps), saves to R2, writes `daily_piece_audio` rows. 20k-char hard cap per piece, 3-attempt retry on transient failures, request-stitching for prosodic continuity. Provider-agnostic TTS normaliser handles Roman numerals + brand pronunciation.
10. **AudioAuditorAgent** — reads `daily_piece_audio` rows, verifies R2 objects exist + file sizes are sane + total chars under cap. Passes/fails without touching git. As of 2026-05-12 (Foundation Fix Task 05) also persists per-issue rows + always-one summary row to `audio_audit_results` (migration 0033) on every `audit()` call. Closed-enum `AudioIssueType` (8 values) maps each branch; persistence is fail-open via `persistError` sentinel that Director logs once via `observer.logError` (no per-row spam). Audio Producer in the same task starts populating the previously always-NULL `daily_piece_audio.duration_seconds` (computed from `byteLength / 12000` per 96 kbps `AUDIO_OUTPUT_FORMAT`) and the new `file_size_bytes` column.
11. **PublisherAgent** — commits to GitHub, piece goes live
12. **LearnerAgent** — learns from reader behaviour and producer signals, writes patterns for future pieces. Three sources: producer (post-publish), self-reflection (Drafter post-publish), reader (engagement-driven).
13. **CategoriserAgent** — assigns 1–3 library categories to each piece post-publish (off-pipeline alarm), strongly biased toward reusing the existing taxonomy. Layered defense (≥75 reuse floor, single retry on empty/all-sub-floor, `Patterns Yet to Cluster` fallback).
14. **ObserverAgent** — logs every pipeline event for the admin dashboard
15. **InteractiveGeneratorAgent** — produces a standalone quiz per piece post-publish (off-pipeline alarm). Teaches the underlying concept, never references the source piece. Owns a produce→audit→revise loop up to 3 rounds; ships as `quality_flag='low'` on max-fail rather than abandoning. On parse-fail (`parseAndValidate*: Claude returned non-JSON output`), the next round routes through a JSON-repair revision (broken-output head quoted back to Claude) instead of re-running the initial prompt blind — added 2026-05-05 after a 2026-05-05 piece burned all 3 rounds on the same unquoted-`concept` defect while the manual admin retrigger succeeded immediately. Root-cause investigation of why Sonnet 4.5 occasionally drops opening quotes on long natural-language values is queued for after Foundation Fix Phase 2.
16. **InteractiveAuditorAgent** — single Claude call judges quizzes across voice / structure-pedagogy / essence-not-reference / factual dimensions. Called internally by InteractiveGenerator each round.
